Cyphers runs on two Solana clusters: **devnet** for development and testing, and **mainnet-beta** for production. The `cluster` option in `CypherClient` controls which cluster your app targets - it configures the Arcium MPC endpoint, the expected token mint, and other protocol-level constants automatically. This page shows you how to switch between clusters and what to watch out for when moving to production.

## Setting the cluster

Pass `cluster` when constructing `CypherClient`. The value must be either `'devnet'` or `'mainnet-beta'`:

<CodeGroup>

```typescript Devnet
import { Connection } from "@solana/web3.js";
import { CypherClient } from "@cypher-zk/sdk";

const client = new CypherClient({
  connection: new Connection("https://api.devnet.solana.com", "confirmed"),
  wallet,
  cluster: "devnet",
});
```

```typescript Mainnet
import { Connection } from "@solana/web3.js";
import { CypherClient } from "@cypher-zk/sdk";

const client = new CypherClient({
  connection: new Connection("https://api.mainnet-beta.solana.com", "confirmed"),
  wallet,
  cluster: "mainnet-beta",
});
```

</CodeGroup>

<Note>
  Always use devnet during development. Devnet lets you airdrop SOL for free and run end-to-end flows without using real funds. Switch to `mainnet-beta` only when you're ready to go live.
</Note>

## RPC endpoints

The public RPC endpoints are suitable for development, but for production you should use a dedicated RPC provider. High-traffic apps benefit significantly from faster and more reliable connections.

| Cluster | Public endpoint | Recommended for |
|---|---|---|
| Devnet | `https://api.devnet.solana.com` | Development and testing |
| Mainnet | `https://api.mainnet-beta.solana.com` | Small-scale production |
| Mainnet (dedicated) | Helius, QuickNode, or Triton | High-traffic production |

```typescript
// Using a dedicated RPC provider on mainnet
const client = new CypherClient({
  connection: new Connection(
    "https://your-endpoint.helius-rpc.com/?api-key=<KEY>",
    "confirmed"
  ),
  wallet,
  cluster: "mainnet-beta",
});
```

<Note>
  The public `getProgramAccounts` endpoint used by `client.markets.all()` is slow on public RPCs. If your app lists many markets, use a dedicated provider and filter aggressively - for example, `{ state: MarketState.Active }` - or move to an off-chain indexer.
</Note>

## Reading the USDC mint address

Cyphers markets use USDC as their settlement currency. The accepted mint address is stored in the protocol's on-chain `GlobalState` account - you should always read it from there rather than hardcoding it.

```typescript
// ✅ Read the mint from GlobalState
const globalState = await client.globalState.fetch();
const acceptedMint = globalState.acceptedMint;

// Use acceptedMint when building ATAs or passing to instruction builders
const ix = await client.marketIx.createIx({
  // ...
  acceptedMint,
});
```

<Warning>
  **Never hardcode the USDC mint address.** The protocol can update the accepted mint via an admin instruction. Apps that hardcode the address will break on a mint rotation and may route funds to the wrong token account. Always read `GlobalState.acceptedMint` at runtime.
</Warning>

## Switching clusters from an environment variable

A common pattern is to drive the cluster choice from an environment variable so the same codebase targets devnet in CI and mainnet in production:

```typescript
import { Connection } from "@solana/web3.js";
import { CypherClient } from "@cypher-zk/sdk";

const isMainnet = process.env.NEXT_PUBLIC_CLUSTER === "mainnet-beta";

const client = new CypherClient({
  connection: new Connection(
    isMainnet
      ? "https://api.mainnet-beta.solana.com"
      : "https://api.devnet.solana.com",
    "confirmed"
  ),
  wallet,
  cluster: isMainnet ? "mainnet-beta" : "devnet",
});
```

## Program ID

The Cyphers program is deployed at the same address on both clusters:

```
CYPHERKLYj1bZtWv7Xut6Snw13ab1DwdJia5ZSEeVeej
```

You don't need to pass `programId` to `CypherClient` unless you're deploying a custom build to a local validator. The SDK exports `PROGRAM_ID` as a constant for use in scripts and tests:

```typescript
import { PROGRAM_ID } from "@cypher-zk/sdk";

console.log(PROGRAM_ID.toBase58());
// CYPHERKLYj1bZtWv7Xut6Snw13ab1DwdJia5ZSEeVeej
```

## Devnet faucet and token setup

To run end-to-end tests on devnet, your wallet needs SOL for transaction fees and a funded USDC token account:

```bash
# Airdrop SOL to your devnet wallet
solana airdrop 1 ~/.config/solana/devnet.json --url devnet

# Create a USDC token account and mint test tokens
spl-token create-account <ACCEPTED_MINT> --url devnet
spl-token mint <ACCEPTED_MINT> 100 --url devnet
```

Replace `<ACCEPTED_MINT>` with the value of `GlobalState.acceptedMint` on devnet. USDC uses 6 decimal places - 1 USDC equals `1_000_000` in raw token units.
