The `@cypher-zk/sdk/react` subpath exports a `CypherProvider` context and a set of TanStack Query-powered hooks that handle all data fetching, cache invalidation, and mutation state for you. This guide shows you how to wire everything together - from provider setup through real-time event subscriptions - in both a Vite SPA and a Next.js App Router project.

## Installation

```bash
npm install @cypher-zk/sdk @solana/web3.js@^1.95.4 @tanstack/react-query
```

Install Solana wallet adapter packages for wallet UI:

```bash
npm install @solana/wallet-adapter-react @solana/wallet-adapter-react-ui @solana/wallet-adapter-wallets
```

<Warning>
  Use `@solana/web3.js` **v1** (e.g. `^1.95.4`). The SDK is not compatible
  with v2. Do not import `@cypher-zk/sdk/node` in any browser code - it will
  break your bundler.
</Warning>

## Provider setup

<Tabs>
  <Tab title="React / Vite">
    Create a `Providers` component that nests `ConnectionProvider`, `WalletProvider`, `QueryClientProvider`, and `CypherProvider`. Place it at the root of your app.

    ```typescript
    // src/providers.tsx
    import { useMemo } from 'react';
    import { Connection } from '@solana/web3.js';
    import { CypherClient } from '@cypher-zk/sdk';
    import { CypherProvider } from '@cypher-zk/sdk/react';
    import {
      ConnectionProvider,
      WalletProvider,
      useWallet,
    } from '@solana/wallet-adapter-react';
    import { WalletModalProvider } from '@solana/wallet-adapter-react-ui';
    import { PhantomWalletAdapter } from '@solana/wallet-adapter-wallets';
    import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
    import '@solana/wallet-adapter-react-ui/styles.css';

    const RPC      = import.meta.env.VITE_RPC_URL ?? 'https://api.devnet.solana.com';
    const wallets  = [new PhantomWalletAdapter()];
    const qc       = new QueryClient({ defaultOptions: { queries: { staleTime: 10_000 } } });

    function CypherShell({ children }: { children: React.ReactNode }) {
      const wallet = useWallet();
      const client = useMemo(() => {
        if (!wallet.publicKey) return null;
        return new CypherClient({
          connection: new Connection(RPC, 'confirmed'),
          wallet: wallet as never, // cast is safe - adapter shape matches SDK Wallet interface
          cluster: 'devnet',
        });
      // Re-create only when the connected public key changes, not on every render
      // eslint-disable-next-line react-hooks/exhaustive-deps
      }, [wallet.publicKey]);

      if (!client) return <>{children}</>;
      return <CypherProvider client={client}>{children}</CypherProvider>;
    }

    export function Providers({ children }: { children: React.ReactNode }) {
      return (
        <ConnectionProvider endpoint={RPC}>
          <WalletProvider wallets={wallets} autoConnect>
            <WalletModalProvider>
              <QueryClientProvider client={qc}>
                <CypherShell>{children}</CypherShell>
              </QueryClientProvider>
            </WalletModalProvider>
          </WalletProvider>
        </ConnectionProvider>
      );
    }
    ```

    Wrap your app entry point:

    ```typescript
    // src/main.tsx
    import React from 'react';
    import ReactDOM from 'react-dom/client';
    import { Providers } from './providers';
    import App from './App';

    ReactDOM.createRoot(document.getElementById('root')!).render(
      <React.StrictMode>
        <Providers>
          <App />
        </Providers>
      </React.StrictMode>,
    );
    ```

    If you see `Buffer is not defined` at runtime, add the polyfill at the top of `providers.tsx`:

    ```typescript
    import { Buffer } from 'buffer';
    if (typeof window !== 'undefined') (window as never).Buffer ??= Buffer;
    ```

    And update `vite.config.ts`:

    ```typescript
    import { defineConfig } from 'vite';
    import { nodePolyfills } from 'vite-plugin-node-polyfills';

    export default defineConfig({
      plugins: [nodePolyfills()],
    });
    ```
  </Tab>

  <Tab title="Next.js App Router">
    All Cyphers components must be **client components** - the SDK uses `@solana/web3.js` which is not SSR-safe. Add `'use client'` to every file that imports from `@cypher-zk/sdk` or `@cypher-zk/sdk/react`.

    <Note>
      Mark every component that touches Cyphers SDK hooks with the `'use client'`
      directive. The `CypherProvider` wrapper itself must live in a client
      component - it cannot be placed in a Server Component or `app/layout.tsx`
      directly.
    </Note>

    **`app/providers.tsx`**

    ```typescript
    'use client';
    import { Buffer } from 'buffer';
    if (typeof window !== 'undefined') (window as never).Buffer ??= Buffer;

    import { useMemo, useState } from 'react';
    import { Connection } from '@solana/web3.js';
    import { CypherClient } from '@cypher-zk/sdk';
    import { CypherProvider } from '@cypher-zk/sdk/react';
    import {
      ConnectionProvider,
      WalletProvider,
      useWallet,
    } from '@solana/wallet-adapter-react';
    import { WalletModalProvider } from '@solana/wallet-adapter-react-ui';
    import { PhantomWalletAdapter } from '@solana/wallet-adapter-wallets';
    import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

    const RPC     = process.env.NEXT_PUBLIC_RPC_URL ?? 'https://api.devnet.solana.com';
    const wallets = [new PhantomWalletAdapter()];

    function CypherShell({ children }: { children: React.ReactNode }) {
      const wallet = useWallet();
      const client = useMemo(() => {
        if (!wallet.publicKey) return null;
        return new CypherClient({
          connection: new Connection(RPC, 'confirmed'),
          wallet: wallet as never,
          cluster: 'devnet',
        });
      // eslint-disable-next-line react-hooks/exhaustive-deps
      }, [wallet.publicKey]);

      if (!client) return <>{children}</>;
      return <CypherProvider client={client}>{children}</CypherProvider>;
    }

    export function Providers({ children }: { children: React.ReactNode }) {
      const [qc] = useState(() => new QueryClient({
        defaultOptions: { queries: { staleTime: 10_000 } },
      }));

      return (
        <ConnectionProvider endpoint={RPC}>
          <WalletProvider wallets={wallets} autoConnect>
            <WalletModalProvider>
              <QueryClientProvider client={qc}>
                <CypherShell>{children}</CypherShell>
              </QueryClientProvider>
            </WalletModalProvider>
          </WalletProvider>
        </ConnectionProvider>
      );
    }
    ```

    **`app/layout.tsx`**

    ```typescript
    import { Providers } from './providers';
    import '@solana/wallet-adapter-react-ui/styles.css';

    export default function RootLayout({ children }: { children: React.ReactNode }) {
      return (
        <html lang="en">
          <body>
            <Providers>{children}</Providers>
          </body>
        </html>
      );
    }
    ```

    **`next.config.js`** - add webpack fallbacks for Node built-ins:

    ```javascript

    module.exports = {
      webpack: (config) => {
        config.resolve.fallback = {
          ...config.resolve.fallback,
          crypto: false,
          fs:     false,
          stream: false,
        };
        return config;
      },
    };
    ```

## Listing markets with `useMarkets`

`useMarkets` returns a paginated list of market accounts, optionally filtered by state or creator. Batch-fetch question text with `fetchMarketQuestions` - questions live in a separate PDA, not on the market account itself.

```typescript
'use client';
import { useEffect, useState } from 'react';
import { useMarkets, useCypherClient } from '@cypher-zk/sdk/react';
import {
  fetchMarketQuestions,
  parseEmbeddedOptions,
  marketPhase,
  MarketState,
} from '@cypher-zk/sdk';

export function MarketList() {
  const client = useCypherClient();
  const { data: markets, isLoading } = useMarkets({ state: MarketState.Active });
  const [questions, setQuestions] = useState<Map<string, string>>(new Map());

  useEffect(() => {
    if (!markets?.length) return;
    fetchMarketQuestions(client, markets).then(setQuestions);
  }, [client, markets]);

  if (isLoading) return <p>Loading markets…</p>;
  if (!markets?.length) return <p>No open markets.</p>;

  return (
    <ul>
      {markets.map(({ publicKey, account }) => {
        const rawQuestion =
          questions.get(publicKey.toBase58()) ??
          account.inlineQuestion ??
          `Market #${account.marketId}`;
        const { displayQuestion } = parseEmbeddedOptions(rawQuestion);
        const phase = marketPhase(account);

        return (
          <li key={publicKey.toBase58()}>
            <a href={`/markets/${account.marketId}`}>
              <span>[{phase}]</span> {displayQuestion}
            </a>
          </li>
        );
      })}
    </ul>
  );
}
```

<Warning>
  Always call `parseEmbeddedOptions(rawQuestion).displayQuestion` before
  rendering MultiOutcome market titles. The raw question contains a
  `[A|B|C]` suffix that looks broken in the UI if rendered directly.
</Warning>

## Placing a bet with `usePlaceBet`

`usePlaceBet` is a TanStack Query mutation hook. It auto-invalidates the market and position caches after a successful bet.

```typescript
'use client';
import { useState } from 'react';
import { usePlaceBet, useMarket } from '@cypher-zk/sdk/react';
import { getMarketOptionLabels, fetchMarketQuestions, marketPhase } from '@cypher-zk/sdk';
import { useWallet } from '@solana/wallet-adapter-react';

export function BetCard({ marketId, rawQuestion }: { marketId: bigint; rawQuestion: string }) {
  const wallet  = useWallet();
  const { data: market } = useMarket(marketId);
  const placeBet = usePlaceBet({
    onSuccess: ({ userKeypair, betIndex, signature }) => {
      // Persist the secret key - no recovery path if lost
      localStorage.setItem(
        `cypher-key-${marketId}-${betIndex}`,
        JSON.stringify(Array.from(userKeypair.privateKey)),
      );
      console.log('Bet placed:', signature);
    },
  });

  const [side, setSide]     = useState(1);
  const [amount, setAmount] = useState('5');

  if (!market) return null;
  if (marketPhase(market) !== 'betting') return <p>Betting is closed.</p>;

  const labels = getMarketOptionLabels(market, rawQuestion);

  return (
    <div>
      <fieldset>
        {labels.map((label, i) => (
          <label key={i}>
            <input
              type="radio"
              checked={side === i}
              onChange={() => setSide(i)}
              disabled={placeBet.isPending}
            />
            {label}
          </label>
        ))}
      </fieldset>

      <input
        type="number"
        value={amount}
        onChange={(e) => setAmount(e.target.value)}
        disabled={placeBet.isPending}
      />

      <button
        disabled={!wallet.publicKey || placeBet.isPending}
        onClick={() =>
          placeBet.mutate({
            payer:       wallet.publicKey!,
            user:        wallet.publicKey!,
            marketId,
            side,
            amountUsdc:  BigInt(Math.round(parseFloat(amount) * 1_000_000)),
            onProgress:  (e) => console.log(e.stage),
          })
        }
      >
        {placeBet.isPending ? 'Placing bet…' : `Bet $${amount} on ${labels[side]}`}
      </button>
    </div>
  );
}
```

## Real-time events with `useMarketEvents`

`useMarketEvents` opens a WebSocket subscription and returns a live array of `CypherEvent` objects. Use it to update bet counts, odds, or resolution status without polling.

```typescript
'use client';
import { useMarketEvents } from '@cypher-zk/sdk/react';

export function ActivityFeed() {
  const events = useMarketEvents();

  return (
    <ul>
      {events.map((event, i) => (
        <li key={i}>
          <strong>{event.name}</strong>{' '}
          {event.name === 'BetPlacedEvent' && (
            <span>- market {event.data.market.toBase58().slice(0, 8)}…</span>
          )}
          {event.name === 'MarketResolvedEvent' && (
            <span>- outcome {event.data.outcome}</span>
          )}
        </li>
      ))}
    </ul>
  );
}
```

<Tip>
  On mobile or flaky connections, WebSocket subscriptions may drop. Use
  `client.events.pollEvents` as a fallback, or combine both: subscribe for
  instant updates and poll every 30 seconds to catch any missed events.
</Tip>

## All available hooks

| Hook | Kind | Purpose |
|---|---|---|
| `useCypherClient()` | context | Read the `CypherClient` from context (throws outside `<CypherProvider>`) |
| `useMarkets(filter?, opts?)` | query | List markets, optionally filtered by state or creator |
| `useMarket(id, opts?)` | query | Fetch a single market by `marketId` |
| `useUserPositions(user, opts?)` | query | All positions for a wallet across all markets |
| `usePosition(market, user, betIndex?, opts?)` | query | Single position for a `(market, user, betIndex)` tuple |
| `useGlobalState(opts?)` | query | Protocol-wide config (fees, accepted mint, admin) |
| `usePlaceBet(opts?)` | mutation | Encrypt and submit a bet |
| `useCreateMarket(opts?)` | mutation | Create a YesNo market |
| `useClaimPayout(opts?)` | mutation | Claim winnings after resolution |
| `useClaimRefund(opts?)` | mutation | Reclaim stake from an unresolved market |
| `useCancelMarket(opts?)` | mutation | Cancel a market before any bets |
| `useFlagResolution(opts?)` | mutation | Flag an incorrect outcome during the challenge window |
| `useFinalizeResolution(opts?)` | mutation | Finalize an undisputed resolution (anyone can call) |
| `useAdminOverrideResolution(opts?)` | mutation | Admin-only: override a disputed outcome |
| `useMarketEvents(opts?)` | subscription | Stream live `CypherEvent[]` from the program |

## Cache tuning

The SDK's query hooks use TanStack Query with sensible defaults. Override them per call if you need tighter or looser freshness:

```typescript
// Poll a market every second during the challenge window
const { data: market } = useMarket(marketId, { refetchInterval: 1_000 });

// Never stale - you'll invalidate manually after your own transactions
const { data: state } = useGlobalState({ staleTime: Infinity });
```

After using raw instruction builders instead of mutation hooks, invalidate caches manually:

```typescript
import { marketKeys, positionKeys } from '@cypher-zk/sdk/react';
import { useQueryClient } from '@tanstack/react-query';

const qc = useQueryClient();
await qc.invalidateQueries({ queryKey: marketKeys.one(marketId) });
await qc.invalidateQueries({ queryKey: positionKeys.byUser(wallet.publicKey) });
```
