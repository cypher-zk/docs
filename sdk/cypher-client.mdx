`CypherClient` is the single object through which all SDK functionality flows. Construct it once with a `Connection` and a `Wallet`, then access every on-chain operation through its typed namespaces. The constructor is **synchronous** - it sets up the Anchor provider and program bindings locally without firing any RPC call.

<Note>
  You do not need to `await` the constructor. Connections to Solana are made lazily, on the first method call that needs them.
</Note>

## Constructor

```typescript
import { CypherClient } from '@cypher-zk/sdk';
import { Connection } from '@solana/web3.js';

const client = new CypherClient({
  connection: new Connection('https://api.devnet.solana.com', 'confirmed'),
  wallet: yourWallet,
  cluster: 'devnet',
});
```

### Options

<ParamField path="connection" type="Connection" required>
  A `@solana/web3.js` `Connection` instance pointing at your target cluster. Pass your own RPC URL for production - public endpoints are rate-limited.
</ParamField>

<ParamField path="wallet" type="Wallet" required>
  A wallet object that implements `publicKey`, `signTransaction`, and `signAllTransactions`. Compatible with Anchor's `Wallet` type. Use `keypairToWallet(keypair)` for server-side scripts, or pass the adapter wallet directly in a React app.

  ```typescript
  import { keypairToWallet } from '@cypher-zk/sdk';
  import { Keypair } from '@solana/web3.js';

  const keypair = Keypair.fromSecretKey(secretKeyBytes);
  const wallet  = keypairToWallet(keypair);
  ```
</ParamField>

<ParamField path="cluster" type='"devnet" | "mainnet" | "localnet"'>
  Selects the known cluster configuration (RPC URL, expected mint, and related defaults). Defaults to `"devnet"`. You can also pass a full `ClusterConfig` object for a custom endpoint.
</ParamField>

<ParamField path="commitment" type="Commitment">
  Solana commitment level. Defaults to `"confirmed"`. Use `"finalized"` when reading state that gates payouts or resolution decisions.
</ParamField>

<ParamField path="programId" type="PublicKey">
  Override the deployed program address. Only needed when testing against a local redeployment with a different keypair. Defaults to `PROGRAM_ID` (`cyphPe923‚Ä¶oh2B`).
</ParamField>

<ParamField path="confirmOptions" type="ConfirmOptions">
  Options forwarded to Anchor's `AnchorProvider` for transaction confirmation (e.g. `skipPreflight`, `maxRetries`).
</ParamField>

## Full initialization example

```typescript
import { CypherClient, keypairToWallet } from '@cypher-zk/sdk';
import { Connection, Keypair } from '@solana/web3.js';

// --- Browser (React + wallet adapter) ---
import { useWallet } from '@solana/wallet-adapter-react';

function buildClient(wallet: ReturnType<typeof useWallet>) {
  return new CypherClient({
    connection: new Connection(
      process.env.NEXT_PUBLIC_RPC_URL!,
      'confirmed'
    ),
    wallet: {
      publicKey: wallet.publicKey!,
      signTransaction: wallet.signTransaction!,
      signAllTransactions: wallet.signAllTransactions!,
    },
    cluster: 'mainnet',
  });
}

// --- Node.js / admin script ---
const keypair = Keypair.fromSecretKey(
  Uint8Array.from(JSON.parse(process.env.KEYPAIR_JSON!))
);
const client = new CypherClient({
  connection: new Connection('https://api.devnet.solana.com'),
  wallet: keypairToWallet(keypair),
  cluster: 'devnet',
});
```

## Client namespaces

Every namespace is pre-bound to the client instance and mirrors the underlying standalone helper functions. Use namespaces in application code; import the helpers directly only in unit tests.

| Namespace | Purpose |
|---|---|
| `client.globalState` | Fetch and cache the protocol `GlobalState` account (fee rates, accepted mint, admin) |
| `client.markets` | Fetch individual markets, list all markets, filter by creator or state |
| `client.marketQuestions` | Fetch the separate `MarketQuestion` PDA introduced in v0.2 |
| `client.positions` | Fetch a user's encrypted position for a given market and bet index |
| `client.lpPositions` | Fetch LP (creator bond / fee) positions |
| `client.events` | Subscribe to on-chain events via WebSocket or poll by signature |
| `client.actions` | High-level end-to-end action methods - the recommended path for most use cases |
| `client.admin` | One-time protocol setup instructions (**Cyphers team only**) |
| `client.compDefs` | Arcium computation definition registration (**Cyphers team only**) |
| `client.marketIx` | Raw instruction builders for market lifecycle (create, cancel, withdraw) |
| `client.bets` | Raw instruction builders for placing bets |
| `client.resolveIx` | Raw instruction builders for market resolution |
| `client.claimIx` | Raw instruction builders for payout and refund claims |
| `client.resolutionIx` | Raw instruction builders for the v0.2 dispute/challenge window |

## Fetch methods

### Global state

```typescript
// Fetch the protocol config (fee rates, accepted mint, market counter)
const globalState = await client.globalState.fetch();

console.log(globalState.protocolFeeRate);   // bigint - basis points
console.log(globalState.acceptedMint);      // PublicKey - USDC mint
console.log(globalState.marketCounter);     // bigint - total markets created
```

### Markets

```typescript
// All markets
const markets = await client.markets.all();

// Single market by ID (u64 as bigint or number)
const market = await client.markets.fetch(42n);

// Markets created by a specific wallet
const myMarkets = await client.markets.byCreator(wallet.publicKey);

// Markets filtered by on-chain state (Active = 0, Resolved = 2, etc.)
import { MarketState } from '@cypher-zk/sdk';
const activeMarkets = await client.markets.byState(MarketState.Active);
```

<Tip>
  For v0.2+ accounts, market questions live in a separate PDA. Batch-fetch them alongside markets using `fetchMarketQuestions` to avoid N+1 RPC calls:

  ```typescript
  import { fetchMarketQuestions } from '@cypher-zk/sdk';

  const markets   = await client.markets.all();
  const pdas      = markets.map(m => m.publicKey);
  const questionMap = await fetchMarketQuestions(client, pdas);

  for (const { publicKey, account } of markets) {
    const question =
      questionMap.get(publicKey.toBase58()) ||
      account.inlineQuestion ||
      `Market #${account.marketId}`;
  }
  ```
</Tip>

### Positions

```typescript
// A user's position on a specific market (betIndex defaults to 0n)
const position = await client.positions.fetch(marketPda, userPublicKey);

// All positions a user holds across every market
const allPositions = await client.positions.byUser(userPublicKey);

// All positions for a given market (resolver / analytics use)
const marketPositions = await client.positions.forMarket(marketPda);
```

### LP positions

```typescript
// The creator's LP position on their market
const lpPosition = await client.lpPositions.fetch(marketPda, creatorPublicKey);

// All LP positions held by a liquidity provider
const providerPositions = await client.lpPositions.byProvider(providerPublicKey);
```

## Account types reference

The following TypeScript interfaces describe the deserialized on-chain accounts returned by the fetch methods above. All numeric fields use `bigint`; all key fields use `PublicKey`.

### `GlobalStateAccount`

<ResponseField name="marketCounter" type="bigint">
  Total number of markets ever created. Used to derive the next market PDA.
</ResponseField>
<ResponseField name="protocolFeeRate" type="bigint">
  Protocol fee in basis points (max `100` = 1%).
</ResponseField>
<ResponseField name="lpFeeRate" type="bigint">
  LP fee in basis points (max `500` = 5%).
</ResponseField>
<ResponseField name="acceptedMint" type="PublicKey">
  The SPL token mint accepted by the protocol (USDC on all clusters).
</ResponseField>
<ResponseField name="admin" type="PublicKey">
  The Cyphers admin authority. Required for override resolution and treasury claims.
</ResponseField>
<ResponseField name="protocolTreasury" type="PublicKey">
  Destination for protocol fee receipts.
</ResponseField>

### `MarketAccount`

<ResponseField name="marketId" type="bigint">
  Unique sequential market ID. Combine with `PROGRAM_ID` to derive the market PDA via `marketPda(marketId)`.
</ResponseField>
<ResponseField name="marketType" type="number">
  `0` = YesNo, `1` = MultiOutcome. Use `marketTypeName(n)` for a display string.
</ResponseField>
<ResponseField name="state" type="number">
  `0` Active ¬∑ `1` Closed ¬∑ `2` Resolved ¬∑ `3` Unresolved ¬∑ `4` PendingResolution. Use `marketStateName(n)` for a display string.
</ResponseField>
<ResponseField name="closeTime" type="bigint">
  Unix timestamp (seconds) after which betting closes.
</ResponseField>
<ResponseField name="outcome" type="number | null">
  Resolved winning outcome index. `null` until resolved.
</ResponseField>
<ResponseField name="totalBetsCount" type="bigint">
  Total number of bets placed on this market.
</ResponseField>
<ResponseField name="creator" type="PublicKey">
  The wallet that created and bonded this market.
</ResponseField>
<ResponseField name="resolver" type="PublicKey">
  The wallet authorized to call `resolveMarket`.
</ResponseField>
<ResponseField name="creatorBond" type="bigint">
  USDC (in lamports) locked as the creator bond (min `MIN_CREATOR_BOND` = $20).
</ResponseField>
<ResponseField name="inlineQuestion" type="string">
  Non-empty only for legacy v1/v2 accounts (577 or 594 bytes). Always `""` for current v3 accounts - use `fetchMarketQuestion` instead.
</ResponseField>

### `EncryptedPositionAccount`

<ResponseField name="user" type="PublicKey">
  The bettor's wallet address.
</ResponseField>
<ResponseField name="market" type="PublicKey">
  The market PDA this position belongs to.
</ResponseField>
<ResponseField name="encryptedAmount" type="Uint8Array">
  32-byte Rescue-encrypted bet amount. Decryptable only by the Arcium MPC network and the user who holds the private key.
</ResponseField>
<ResponseField name="encryptedSide" type="Uint8Array">
  32-byte Rescue-encrypted outcome index.
</ResponseField>
<ResponseField name="entryOdds" type="bigint">
  Odds at the time the bet was placed, scaled by `ODDS_SCALE` (1e9).
</ResponseField>
<ResponseField name="netAmount" type="bigint">
  Bet amount after protocol and LP fees were deducted (USDC lamports).
</ResponseField>
<ResponseField name="betIndex" type="bigint">
  Per-user bet counter. Allows placing multiple bets on the same market.
</ResponseField>
<ResponseField name="claimed" type="boolean">
  `true` once a payout or refund has been claimed for this position.
</ResponseField>

## Using raw instruction builders

The namespace-based instruction builders let you compose, simulate, or bundle instructions before sending them. All builders return `Promise<TransactionInstruction>`.

```typescript
import { sendIx } from '@cypher-zk/sdk';

// Build a raw cancel instruction
const cancelIx = await client.marketIx.cancelIx({
  creator: wallet.publicKey,
  marketId: 7n,
  acceptedMint: usdcMint,
});

// Simulate before sending
const simulation = await client.connection.simulateTransaction(
  new Transaction().add(cancelIx)
);

// Send when ready
const signature = await sendIx(client, cancelIx);
```

<Warning>
  When you bypass the high-level `client.actions.*` methods and send instructions manually, you are responsible for invalidating any TanStack Query caches. See the [React Hooks](/sdk/react-hooks) page for the query-key factory reference.
</Warning>
